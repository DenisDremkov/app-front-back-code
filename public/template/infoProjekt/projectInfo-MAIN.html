Пароль администратора - 'admin'

Краткая информация по проекту.

Есть три основных интерфейса (контроллера):

  1. Аутентификация.
  2. Пользователь - страница с товарами. (4 дочерних контроллера)
  3. Администратор - страница с функционалом администратора. (3 дочерних контроллера)

Все модели находятся в контроллерах, в контроллерах происходят только манипуляции с представлениями, вся логика перенесена в сервисы. Каждый контроллер имеет свой сервис. Методы повторно, за редким исключением, практически не используются.

Большинство кода написано для возможного расширения (добавления новой категории товара), например ползунки с макс и мин значениями характеристик товаров, если в бд добавить новые свойства, то на клиенте не надо ничего переделывать.
